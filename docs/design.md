# Stella Trader v2 設計書

## 概要

仮想通貨自動売買システムの全面再構築。既存のJavaScript実装をPython(ボットエンジン) + TypeScript(ダッシュボード)のハイブリッド構成に移行し、バックテスト基盤・安全機構・状態管理を根本から再設計する。

## 背景と動機

### 既存システムの問題点（v1 ポストモーテムより）

1. **状態管理と実残高の不同期** → ¥38,000の損失を引き起こした致命的な問題
2. **複数戦略が同じペアで干渉** → ポジション管理が戦略ごとに独立しており統合管理されていない
3. **バックテスト基盤がない** → 戦略の事前検証なしで本番投入
4. **JavaScript + テストなし** → 型安全性の欠如とリグレッション防止策の不在
5. **エラーハンドリング・リトライ不足** → API障害時の復旧が不完全

## 技術スタック

### ボットエンジン（Python）

| 要素 | 選定 | 理由 |
|------|------|------|
| 言語 | Python 3.12+ | 金融ライブラリが最も豊富 |
| 取引所API | ccxt | 複数取引所対応、Bybitサポート |
| テクニカル指標 | pandas-ta | TA-Libより導入が容易、pandas統合 |
| データ処理 | pandas + numpy | 時系列データの標準ツール |
| バックテスト | vectorbt | 高速ベクトル化バックテスト |
| 設定管理 | pydantic | 型安全な設定バリデーション |
| テスト | pytest + pytest-asyncio | 非同期テスト対応 |
| タスクスケジューラ | APScheduler | cron的な定期実行 |
| 通知 | discord.py webhooks | Discord通知 |

### ダッシュボード（TypeScript）

| 要素 | 選定 | 理由 |
|------|------|------|
| フレームワーク | Next.js 15 | 既存資産の活用 |
| グラフ | Recharts | 既存資産の活用 |
| スタイル | Tailwind CSS | 既存資産の活用 |
| データ連携 | REST API (FastAPI) | ボットからリアルタイムデータ取得 |

## 設計方針

### 1. 状態管理の一元化

v1の最大の問題であった状態と実残高の乖離を解決する。

- **ポートフォリオマネージャー**がすべてのポジションを統合管理
- 注文前に必ず実残高をAPI経由で確認
- 状態ファイルと実残高の定期同期（1分間隔）
- 乖離検知時は即座にアラート + 取引停止

### 2. 戦略の独立性と統合管理

- 各戦略は独立したクラスとして実装（Strategy基底クラス継承）
- ポジション管理は戦略レベルではなくポートフォリオレベルで統合
- 戦略間の干渉を防ぐため、ポートフォリオマネージャーが注文をバリデーション

### 3. バックテスト駆動開発

- すべての戦略はバックテストで検証してから実運用
- バックテスト → ペーパートレード → 少額実運用 → 本番のパイプライン
- バックテスト結果の自動レポート生成

### 4. 安全機構の強化

| 機能 | v1 | v2 |
|------|----|----|
| 日次損失上限 | ¥5,000固定 | 口座残高の%ベース（設定可能） |
| 急変動検知 | 5分2%固定 | ATRベースの動的閾値 |
| 注文前残高確認 | なし | 必須 |
| キルスイッチ | なし | 即時全ポジション決済 |
| 1トレードリスク | なし | 口座残高の1-2% |
| 最大ドローダウン | なし | 20%で自動停止 |

### 5. 取引所抽象化

- ccxtを使用した取引所抽象化レイヤー
- 将来的にBinanceなど他取引所への切り替えが容易
- テスト時はモック取引所を注入可能

## 初期戦略: トレンドフォロー

### 概要

EMAクロスオーバー + ADXフィルターによるトレンドフォロー戦略。

### シグナル条件

**買いシグナル:**
- EMA(9) > EMA(21)（ゴールデンクロス）
- ADX(14) > 25（十分なトレンド強度）
- 出来高フィルター（平均出来高の1.5倍以上）

**売りシグナル:**
- EMA(9) < EMA(21)（デッドクロス）
- またはトレーリングストップ発動
- または固定ストップロス発動

### リスク管理パラメータ

- ストップロス: ATR(14) × 2
- トレーリングストップ: ATR(14) × 3（トリガー後ATR × 1.5で追従）
- ポジションサイズ: 口座残高の2%リスク / ストップロス幅
- 最大ポジション数: 3

### 対象ペア

- BTC/USDT（メイン）
- ETH/USDT（サブ）

## データフロー

```
                                                                           ↓

```
